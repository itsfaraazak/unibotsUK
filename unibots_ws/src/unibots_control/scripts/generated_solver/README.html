c/src/cpg_workspace.c

Contains the static allocation of all C variables declared in c/include/cpg_workspace.h.

c/src/cpg_solve.c

Contains the definitions of all functions declared in c/include/cpg_solve.h.

c/src/cpg_example.c

Contains example C code that initializes the problem data and solves a problem instance. See Using Generated Code below for compilation instructions.

c/CMakeLists.txt

Contains instructions for build file generation inside the c/build/ directory using CMake. The cpg_example executable and the cpg static library (used by the Python wrapper) are configured here.

c/solver_code/

Contains all source files from the chosen numerical solver (OSQP). Also see the OSQP documentation.

cpp/include/cpg_module.hpp

Declares the pybind11 interface that exposes the generated C solver to Python. Defines the module entry point and the signatures of all bound functions.

cpp/src/cpg_module.cpp

Implements the pybind11 bindings declared in cpg_module.hpp. Compiled together with the cpg static library via CMake (pybind11_add_module) into a Python extension module.

CMakeLists.txt

Root CMake entry point for the generated solver package. Orchestrates the build by adding the c/ subdirectory (C solver and static library) and then directly building the pybind11 Python extension module from cpp/src/cpg_module.cpp. Invoked by pyproject.toml via scikit-build-core.

pyproject.toml

Build configuration for scikit-build-core. Drives CMake to compile the pybind11 extension module. Invoked automatically by CVXPYgen; can also be run manually via pip install --no-build-isolation --no-deps ..

cpg_solver.py

A custom CVXPY solve method that routes prob.solve(method='CPG') through the Python extension module to the generated C solver.

Using Generated Code

CVXPY Interface

Register and call the custom solve method:

from generated_solver.cpg_solver import cpg_solve
prob.register_solve('CPG', cpg_solve)

<param>.value = ...
prob.solve(method='CPG')
print(<var>.value)

Here, <param> and <var> are placeholders for parameter and variable names, respectively.

Example Executable

Build and execute cpg_example as:

cd generated_solver/c/build
cmake ..
cmake --build . --target cpg_example
./cpg_example
cd generated_solver\c\build
cmake ..
cmake --build . --target cpg_example --config release
Release\cpg_example

Embed in C/C++ project

Copy the following into a subdirectory <subdir> of your project:

generated_solver/c/CMakeLists.txt
generated_solver/c/src/*
generated_solver/c/include/*
generated_solver/c/solver_code/*

In your project's top-level CMakeLists.txt, add:

add_subdirectory(<subdir>)
include_directories(... ${cpg_include})

where ... is a placeholder for your existing include directories and the list of include directories of the generated code ${cpg_include} is exposed by generated_solver/c/CMakeLists.txt. Add ${cpg_head} and ${cpg_src} to any targets that use the generated code.
